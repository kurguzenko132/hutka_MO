# STEP 10 — Hypotheses workflow

В этом шаге добавлен полноценный раздел «Гипотезы».

## Что появилось

- список гипотез `/hypotheses`;
- создание гипотезы `/hypotheses/new`;
- карточка гипотезы `/hypotheses/[id]`;
- статусы проверки:
  - Новая;
  - В проверке;
  - Подтверждается;
  - Не подтверждается;
  - Нужно больше данных;
  - Закрыта;
- уровни уверенности:
  - Низкая;
  - Средняя;
  - Высокая;
- поля проверки:
  - как проверяем;
  - метрика успеха;
  - данные за;
  - данные против;
  - результат проверки;
  - следующее действие;
- связи гипотезы с:
  - контактами;
  - инсайтами;
  - кампаниями;
  - опросниками;
- блок «Гипотезы в проверке» на dashboard.

## Что обновить в Supabase

Открой Supabase → SQL Editor и выполни обновленный файл:

```text
supabase/schema.sql
```

В этом шаге добавлены новые поля в `hypotheses`:

```sql
category
test_method
success_metric
confidence
updated_at
```

И новые link-таблицы:

```sql
hypothesis_leads
hypothesis_insights
hypothesis_campaigns
hypothesis_surveys
```

Также добавлен view:

```sql
view_hypothesis_summary
```

## Как проверить

1. Открой `/hypotheses`.
2. Нажми «Добавить гипотезу».
3. Заполни название, способ проверки и метрику успеха.
4. Привяжи контакты, инсайты, кампании или опросники.
5. Сохрани гипотезу.
6. Открой карточку гипотезы.
7. Обнови статус, данные за/против и результат.
8. Открой `/dashboard` и проверь блок «Гипотезы в проверке».
