# Step 11 — реальные отчеты

В этом шаге добавлен полноценный раздел `/reports`, который собирает рабочую сводку по запуску Hutka.

## Что добавлено

- `lib/reports.ts` — агрегация данных для еженедельного отчета.
- `components/reports/copy-report-button.tsx` — кнопка копирования отчета для команды.
- Обновлена страница `/reports`.
- В отчет попадают:
  - контакты;
  - готовые к пилоту;
  - активные участники;
  - ответы на опросы;
  - просроченные действия;
  - воронка по стадиям;
  - лучшие каналы;
  - лучшие ниши;
  - кампании;
  - опросники;
  - инсайты;
  - гипотезы;
  - рекомендации на следующий шаг;
  - готовый текст для копирования в Telegram/чат команды.

## Обновление Supabase

В Supabase SQL Editor выполни обновленный файл:

```text
supabase/schema.sql
```

В этом шаге добавлены views:

```text
view_report_overview
view_report_stage_distribution
view_report_source_distribution
view_report_niche_distribution
```

Сейчас приложение строит отчет через TypeScript-агрегацию из существующих таблиц, а views добавлены как фундамент для будущей более быстрой аналитики.

## Проверка

После деплоя открой:

```text
/reports
```

Проверь:

1. Метрики сверху отображаются.
2. Воронка по стадиям строится.
3. Каналы и ниши показываются.
4. Кампании, опросники, инсайты и гипотезы кликабельны.
5. Кнопка «Скопировать для команды» копирует готовый отчет.
