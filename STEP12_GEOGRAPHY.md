# STEP 12 — Geography analytics

Этот шаг добавляет реальную географическую аналитику для Hutka.

## Что добавлено

- страница `/geography` теперь работает от данных Supabase;
- карточки по городам;
- таблица всех городов;
- метрики по контактам, мастерам, салонам, пилоту и горячим контактам;
- лучшие связки `город + ниша`;
- расчет готовности города к запуску;
- рекомендация, где запускать первую волну;
- ссылки из географии в список контактов по городу;
- fallback на демо-данные, если Supabase еще не настроен.

## Что сделать в Supabase

Открой:

```text
Supabase → SQL Editor
```

И выполни обновленный файл:

```text
supabase/schema.sql
```

## Проверка

После деплоя открой:

```text
/geography
```

Проверь:

1. Сверху отображаются общие метрики.
2. Есть рекомендация запуска.
3. Города отсортированы по готовности к пилоту.
4. Блок `Лучшие связки город + ниша` показывает реальные данные.
5. В таблице городов есть ссылка `Контакты`.
