# STEP 14 — Улучшенный раздел «Люди»

В этом шаге раздел `/people` стал рабочим инструментом для обработки базы контактов, а не просто таблицей.

## Что добавлено

- Реальный поиск по контактам:
  - имя;
  - город;
  - ниша;
  - источник;
  - стадия;
  - Instagram;
  - Telegram;
  - телефон;
  - email;
  - заметки;
  - теги.

- Фильтры через URL-параметры:
  - тип;
  - город;
  - ниша;
  - стадия;
  - источник;
  - приоритет;
  - тег.

- Фильтры работают и с Supabase, и с демо-данными.

- Добавлен отдельный компонент:

```text
components/people/people-filters.tsx
```

- Таблица контактов стала удобнее:
  - показывает теги;
  - показывает контактный канал;
  - добавлены быстрые действия;
  - есть пустое состояние, если контакты не найдены.

- Быстрые действия в таблице:
  - открыть карточку;
  - редактировать контакт;
  - создать задачу по контакту.

- Страница создания задачи теперь принимает параметр:

```text
/tasks/new?leadId=<id>
```

И автоматически выбирает контакт в форме задачи.

- Добавлен экспорт контактов в CSV:

```text
/people/export
```

## Что проверить после деплоя

Открой:

```text
/people
```

Проверь сценарии:

1. Ввести текст в поиск.
2. Отфильтровать по городу.
3. Отфильтровать по стадии.
4. Отфильтровать по тегу.
5. Нажать «Сбросить».
6. Открыть карточку контакта из таблицы.
7. Нажать иконку редактирования.
8. Нажать иконку создания задачи — контакт должен быть выбран в форме.
9. Нажать «Экспорт» — должен скачаться CSV-файл.
