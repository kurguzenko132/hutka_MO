# STEP 26 — Финальная UX-полировка интерфейса

Этот шаг делает Hutka удобнее для ежедневной работы на desktop и мобильных экранах.

## Что добавлено

- Рабочее мобильное меню вместо неактивной кнопки.
- Нижняя мобильная навигация для быстрых разделов:
  - Главная;
  - Люди;
  - Задачи;
  - Уведомления.
- Мобильный drawer со всеми разделами приложения.
- Счетчик уведомлений в мобильном меню и нижней навигации.
- Поиск в верхней панели теперь ведет в `/people?q=...`.
- Главный контент получил корректный `id="content"` и skip-link для доступности.
- Улучшен `PageHeader`: больше похож на единый рабочий блок, лучше адаптируется на телефоне.
- На dashboard добавлен блок **«Быстрый старт»**.
- Кнопки в hero/focus-блоке dashboard теперь нормально отображаются на мобильном.
- Добавлены focus-visible стили для клавиатурной навигации.
- Добавлены safe-area отступы для мобильной нижней навигации.

## Проверка после деплоя

Открой сайт на телефоне или в DevTools mobile mode:

```text
/dashboard
/people
/tasks
/notifications
```

Проверь:

1. Кнопка меню открывает боковую панель.
2. Нижняя навигация работает.
3. Счетчик уведомлений отображается.
4. Поиск сверху переводит в `/people` с query-параметром.
5. Dashboard показывает блок «Быстрый старт».
6. Контент не перекрывается нижней навигацией.
7. Кнопки и карточки нормально выглядят на мобильной ширине.
