# Step 33 — редактор готовых паков вопросов

## Что добавлено

- Раздел `/settings/question-packs` для управления готовыми паками вопросов.
- Создание нового пака вопросов из интерфейса.
- Редактирование названия, короткого названия, аудитории, статуса, бейджа и описания пака.
- Карточка пака `/settings/question-packs/[id]`.
- Добавление вопросов в пак.
- Редактирование вопросов, типа вопроса, обязательности, порядка и вариантов ответа.
- Удаление вопросов и паков.
- Готовые паки теперь берутся из Supabase, а не только из кода.
- Если Supabase не настроен, используется demo fallback со стандартными паками.
- В карточке контакта блок «Готовые паки вопросов» теперь использует редактируемые паки.

## Новые таблицы Supabase

```text
question_packs
question_pack_questions
```

## Что сделать после замены файлов

1. Обновить Supabase:

```text
Supabase → SQL Editor → выполнить supabase/schema.sql
```

2. Установить зависимости и проверить сборку:

```bash
pnpm install
pnpm build
```

3. Закоммитить:

```bash
git add .
git commit -m "Add editable question pack editor"
git push
```

## Как проверить

1. Войти как admin.
2. Открыть `/settings/question-packs`.
3. Создать новый пак вопросов.
4. Открыть карточку пака.
5. Добавить 2–3 вопроса.
6. Открыть карточку контакта `/people/[id]`.
7. Найти блок «Готовые паки вопросов».
8. Создать ссылку из нового пака.
9. Открыть `/q/[token]` в приватном окне.
10. Заполнить ответы и проверить, что они появились в карточке контакта.

## Важное правило

Системную роль пользователя менять через паки нельзя. Паки вопросов — это только шаблоны анкет для маркетинговой работы с контактами.
