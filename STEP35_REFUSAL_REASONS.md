# STEP 35 — Причины отказа и аналитика отказов

## Что добавлено

- Справочник причин отказа: `/settings/refusal-reasons`.
- Блок «Причина отказа» в карточке контакта.
- Перевод контакта в стадию `Отказ` только через форму с причиной.
- Сохранение причины отказа, комментария и даты отказа в `leads`.
- Запись события в историю контакта.
- Очистка причины отказа, если контакт вернулся в работу.
- Аналитика отказов на dashboard.
- Блок причин отказов в недельном отчете.
- Текст отчета для команды теперь включает топ причин отказа.

## Новые таблицы / поля Supabase

Добавлена таблица:

```sql
refusal_reasons
```

В `leads` добавлены поля:

```sql
refusal_reason_id
refusal_reason
refusal_comment
refused_at
```

Добавлен view:

```sql
view_refusal_reason_distribution
```

## Как проверить

1. Обнови Supabase через `supabase/schema.sql`.
2. Открой `/settings/refusal-reasons` под admin.
3. Проверь список стандартных причин отказа.
4. Открой карточку контакта `/people/[id]`.
5. В блоке «Причина отказа» выбери причину и комментарий.
6. Нажми «Зафиксировать отказ».
7. Контакт должен перейти в стадию `Отказ`.
8. В истории контакта должна появиться запись.
9. На `/dashboard` появится блок аналитики отказов.
10. На `/reports` появится блок причин отказов.
