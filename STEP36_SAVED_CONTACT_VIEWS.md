# STEP 36 — сохраненные виды и умные фильтры контактов

## Что добавлено

- В разделе `/people` появился блок **«Быстрые виды»**.
- Добавлены умные выборки:
  - Горячие контакты;
  - Готовы к пилоту;
  - Нужен follow-up;
  - Без ответа;
  - Отказы;
  - Без следующего шага.
- Фильтр `view` работает через URL, например:
  - `/people?view=hot`
  - `/people?view=followup`
  - `/people?view=no-next-step`
- Можно сохранить текущий набор фильтров как личный вид.
- Сохраненные виды привязаны к конкретному пользователю.
- Можно удалить свой сохраненный вид.
- Добавлена таблица `saved_lead_views`.

## Как применить

1. Заменить файлы из архива в проекте.
2. Выполнить обновленный `supabase/schema.sql` в Supabase SQL Editor.
3. Проверить сборку:

```bash
pnpm install
pnpm build
```

4. Закоммитить и отправить на GitHub:

```bash
git add .
git commit -m "Add saved contact views and smart filters"
git push
```

## Проверка после деплоя

Открой:

```text
/people
```

Проверь:

1. Блок **«Быстрые виды»** отображается сверху.
2. Нажми **«Горячие контакты»** — URL станет `/people?view=hot`, список отфильтруется.
3. Нажми **«Нужен follow-up»** — отобразятся контакты с просроченной датой или тегом «Вернуться позже».
4. Примени обычные фильтры, например город + ниша.
5. Введи название и нажми **«Сохранить»**.
6. Сохраненный вид появится в блоке **«Мои сохраненные виды»**.
7. Нажми на сохраненный вид — фильтры применятся снова.
8. Удали сохраненный вид.

## Важное

Сохраненные виды — персональные. Каждый маркетолог видит только свои сохраненные фильтры.
