# STEP 37 — Карточка контакта: следующий шаг и рабочий фокус

## Что добавлено

В этом шаге карточка контакта стала более управленческой. Теперь сверху карточки появляется блок **«Что делать дальше»**, который помогает маркетологу не терять контакт и сразу понимать, какое действие нужно выполнить.

Добавлено:

- автоматический расчет статуса контакта;
- подсветка просроченного follow-up;
- подсветка контактов без следующего шага;
- подсчет открытых и просроченных задач по контакту;
- рекомендация следующего действия на основе стадии;
- быстрый выбор даты follow-up;
- быстрое создание задачи из рекомендации;
- визуальные причины, почему контакт требует внимания.

## Где смотреть

Открой карточку контакта:

```text
/people/[id]
```

Сверху после шапки контакта появится новый блок:

```text
Что делать дальше
```

## Как работает логика

Hutka анализирует:

- стадию контакта;
- приоритет / score;
- следующий шаг;
- дату следующего контакта;
- открытые задачи;
- просроченные задачи;
- причину отказа;
- текущие теги и статус.

После этого система показывает один из статусов:

```text
Контакт под контролем
Follow-up сегодня
Follow-up просрочен
Нет даты следующего контакта
Горячий контакт без закрепленного действия
Контакт в отказе
```

## Рекомендации по стадиям

Примеры автоматических рекомендаций:

```text
Найден → Отправить первое сообщение и зафиксировать реакцию
Написал → Сделать follow-up и уточнить интерес
Ответил → Отправить диагностическую анкету из готового пака
Опрос → Проверить ответы и предложить следующий шаг
Заинтересован → Назначить короткий созвон или пилот
Тест → Помочь пройти пилот и собрать фидбек
Активен → Собрать отзыв, кейс или рекомендацию
Отказ → Зафиксировать причину и дату возврата
```

## Быстрые даты

В карточке доступны быстрые кнопки:

```text
Сегодня
Завтра
Через 3 дня
Через неделю
Через 2 недели
```

При нажатии Hutka обновляет следующий шаг и дату follow-up.

## Что проверить после деплоя

1. Открой `/people/[id]`.
2. Проверь блок **«Что делать дальше»**.
3. Поставь быстрый follow-up на завтра.
4. Создай задачу из рекомендации.
5. Проверь, что задача появилась в карточке и в `/tasks`.
6. У контакта с просроченной датой должен появиться красный статус.
7. У контакта без даты должен появиться желтый статус.
