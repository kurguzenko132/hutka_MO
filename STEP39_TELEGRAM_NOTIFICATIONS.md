# STEP 39 — Telegram-уведомления команды

## Что добавлено

- Раздел `/settings/telegram` для проверки Telegram-интеграции.
- В профиле каждого маркетолога появились поля:
  - `Telegram chat ID`;
  - переключатель `Получать Telegram-уведомления`;
  - кнопка тестового сообщения себе.
- Администратор видит список получателей в `/settings/telegram`.
- Администратор может отправить тестовое сообщение всей команде.
- Когда человек отвечает на персональную анкету `/q/[token]`, Hutka отправляет уведомление всем включенным получателям.
- Добавлен endpoint для ежедневного дайджеста:
  - `/api/telegram/digest?secret=...`
- Добавлены env-переменные:
  - `TELEGRAM_BOT_TOKEN`;
  - `TELEGRAM_DIGEST_SECRET`.

## Как подключить Telegram

1. Открой Telegram и найди `@BotFather`.
2. Создай нового бота.
3. Скопируй token.
4. В Vercel добавь переменную:

```env
TELEGRAM_BOT_TOKEN=токен_бота
```

5. Добавь секрет для endpoint дайджеста:

```env
TELEGRAM_DIGEST_SECRET=любой_сложный_секрет
```

6. Сделай redeploy проекта.
7. Каждый маркетолог должен написать любое сообщение боту.
8. Получи chat ID через:

```text
https://api.telegram.org/botTOKEN/getUpdates
```

9. В Hutka открой `/profile`, вставь chat ID и включи уведомления.
10. Нажми «Отправить тест в Telegram».

## Что проверить

1. Открыть `/profile`.
2. Заполнить `Telegram chat ID`.
3. Включить уведомления.
4. Отправить тест себе.
5. Открыть `/settings/telegram` под admin.
6. Проверить список получателей.
7. Отправить тест команде.
8. Создать персональную анкету в карточке контакта.
9. Открыть ссылку `/q/[token]` в приватном окне и ответить.
10. Проверить, что в Telegram пришло уведомление о новом ответе.

## Endpoint ежедневного дайджеста

Можно вызвать вручную:

```text
/api/telegram/digest?secret=TELEGRAM_DIGEST_SECRET
```

Если нужен автоматический запуск, добавь Vercel Cron позже. Пока endpoint готов, но cron не включен принудительно, чтобы не слать лишние уведомления без контроля.
