# Hutka — шаг 4: вход на главной и формы MVP

## Что изменено

1. Главная страница `/` теперь показывает экран входа, а не промо-лендинг.
2. `/login` использует тот же экран входа.
3. Кнопка входа ведет в демо-dashboard `/dashboard`.
4. Добавлены страницы-заготовки:
   - `/people/new` — добавление контакта;
   - `/tasks/new` — создание задачи;
   - `/surveys/new` — создание опроса;
   - `/campaigns/new` — создание кампании.
5. Кнопки «Добавить контакт», «Создать задачу», «Создать опрос», «Создать кампанию» теперь ведут на соответствующие страницы.
6. Добавлены базовые UI-компоненты:
   - `Textarea`;
   - `Select`;
   - `FormSection`;
   - `Field`.
7. В проект добавлен `vercel.json`, который фиксирует Next.js framework и команды Vercel без ручного output directory.

## Проверка после деплоя

Открой:

- `/` — должен быть вход;
- `/login` — должен быть вход;
- `/dashboard` — dashboard;
- `/people/new` — форма добавления контакта;
- `/tasks/new` — форма создания задачи;
- `/surveys/new` — форма создания опроса;
- `/campaigns/new` — форма создания кампании.

## Важно

Пока формы не сохраняют данные в Supabase. Это следующий этап: подключение CRUD и настоящей авторизации.
