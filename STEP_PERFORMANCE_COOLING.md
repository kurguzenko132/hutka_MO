# Performance cooling fix

Цель: уменьшить нагрев устройства при работе с Hutka на Vercel.

Что изменено:

- Убран тяжелый фон body с несколькими radial-gradient.
- Убраны backdrop-blur эффекты у topbar, sidebar, mobile menu и PageHeader.
- Глобально ограничены длинные transition и hover-transform.
- Сильно облегчены тени в Tailwind config.
- Упрощены графики Dashboard: вместо тяжелых декоративных карточек и мини-баров используются простые progress bars.
- Упрощен FunnelOverview: вместо нескольких декоративных столбиков на каждую стадию — один легкий прогресс.
- В FunnelBoard карточка контакта мемоизирована через React.memo.
- В FunnelBoard убраны hover translate эффекты на карточках.
- Для колонок воронки добавлена базовая оптимизация рендера через performance-contain CSS-класс.
- Для основных навигационных Link отключен prefetch, чтобы Next.js не подгружал десятки страниц заранее при открытом интерфейсе.
- В next.config.ts отключены productionBrowserSourceMaps и включена оптимизация импортов lucide-react.

Проверить после деплоя:

1. /dashboard — прокрутка должна быть легче, графики проще.
2. /people — переходы не должны греть устройство.
3. /people/[id] — действия в карточке не должны вызывать зависания.
4. /funnels — перетаскивание и скролл должны быть легче.
5. Chrome Task Manager: вкладка Hutka не должна постоянно держать высокий CPU в простое.

Если вкладка продолжит греться в простое, следующий шаг — профилирование конкретной страницы через Chrome DevTools Performance и поиск компонента, который постоянно перерисовывается.
