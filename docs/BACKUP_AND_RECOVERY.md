# Hutka — резервные копии и восстановление

## Как сделать бэкап

1. Войти в Hutka под ролью `admin`.
2. Открыть `/launch`.
3. Нажать «Скачать бэкап JSON».
4. Сохранить файл в безопасное место.

Файл содержит рабочие таблицы Hutka: контакты, задачи, опросы, ответы, кампании, инсайты, гипотезы, настройки, справочники и связи.

## Когда делать бэкап

- Перед массовым импортом CSV.
- Перед выполнением нового `supabase/schema.sql`.
- Перед изменением ролей пользователей.
- Перед чисткой дублей.
- Раз в неделю во время активного запуска.

## Как восстановить данные

Автоматическое восстановление из JSON пока не добавлено намеренно: это опасная операция. Восстановление нужно делать вручную через Supabase SQL/API после проверки файла.

Рекомендуемый порядок:

1. Создать новый Supabase-проект или отдельную тестовую базу.
2. Выполнить актуальный `supabase/schema.sql`.
3. Проверить структуру JSON-бэкапа.
4. Импортировать справочники: sources, funnel_stages, tags.
5. Импортировать leads.
6. Импортировать связи и рабочие таблицы.
7. Проверить отчеты, карточки контактов и воронки.

## Минимальный аварийный план

Если данные в рабочей базе повреждены:

1. Не делайте новых массовых действий.
2. Скачайте текущий бэкап для анализа.
3. Откройте последний корректный бэкап.
4. Восстановите данные сначала в тестовой базе.
5. Только после проверки переносите данные в production.
