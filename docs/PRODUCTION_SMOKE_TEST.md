# Hutka — production smoke-test

Этот чеклист нужно проходить после каждого крупного деплоя на Vercel.

## 0. Локальная проверка перед деплоем

```bash
nvm use
pnpm check
pnpm smoke:local
BASE_URL=https://hutka-mo.vercel.app pnpm smoke:url
```

`pnpm smoke:local` ожидает уже собранный `.next`, поднимает `next start` на `PORT=3100` по умолчанию и проверяет ключевые публичные/protected/API маршруты.
`pnpm smoke:url` проверяет деплой без входа и требует, чтобы `/api/health` вернул `supabase-public-env`, `service-role` и `node-runtime` со статусом `ok`. Если один из этих пунктов не `ok`, он появится в `blockers`, а endpoint должен вернуть `status: "needs_attention"`.

## 1. Базовая доступность

- Открыть последний deployment через кнопку **Visit** в Vercel.
- Проверить `/` — должен открыться экран входа.
- Проверить `/api/health` — должен вернуть JSON со статусом приложения.
- В `/api/health` проверки `supabase-public-env`, `service-role` и `node-runtime` должны быть `ok`.
- В `/api/health` массив `blockers` должен быть пустым.
- Проверить `/api/version` — должен вернуть версию проекта и Next.js.

## 2. Авторизация

- Открыть `/dashboard` в приватном окне без входа.
- Должен произойти редирект на `/login`.
- Войти через пользователя Supabase.
- После входа должен открыться dashboard.
- Нажать «Выйти» и убедиться, что сессия завершена.

## 3. Роли

- Под admin открыть `/settings`, `/quality`, `/launch`, `/qa`.
- Под marketer убедиться, что admin-разделы скрыты.
- Под viewer убедиться, что кнопки создания, редактирования, удаления и смены стадии недоступны на списках, карточке контакта, `/funnels`, detail-страницах кампаний/опросов/инсайтов/гипотез и пустом экране уведомлений.

## 4. Контакты

- Создать контакт через `/people/new`.
- Открыть карточку контакта.
- Изменить стадию.
- Добавить касание.
- Создать задачу по контакту.
- Проверить, что события появились в истории.

## 5. Опросники

- Создать опрос.
- Добавить вопросы.
- Открыть публичную ссылку `/s/[slug]` без авторизации.
- Отправить ответ.
- Проверить ответ в карточке опроса.
- Для ответа, привязанного к контакту, создать персональную анкету `/q/[token]` из карточки контакта и проверить ответ там.

## 6. Кампании и аналитика

- Создать кампанию.
- Добавить контакт в кампанию.
- Сохранить вывод по кампании.
- Проверить `/dashboard`, `/reports`, `/geography`, `/funnels`.

## 7. Бэкап

- Под admin открыть `/backup/export`.
- Скачать JSON-файл.
- Проверить, что файл открывается и содержит основные таблицы.

## 8. Финальные команды перед релизом

```bash
pnpm install
pnpm check
pnpm smoke:local
BASE_URL=https://hutka-mo.vercel.app pnpm smoke:url
git add .
git commit -m "Prepare Hutka v1.0 MVP"
git push
```
