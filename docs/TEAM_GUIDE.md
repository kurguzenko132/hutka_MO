# Hutka — инструкция для команды

## Роли

### Admin

Администратор управляет настройками, пользователями, справочниками, качеством MVP и запуском команды. Только admin должен менять роли, выполнять SQL-обновления и скачивать резервные копии.

### Marketer

Маркетолог работает с контактами, задачами, воронками, опросами, кампаниями, инсайтами и гипотезами. Это основная рабочая роль для ежедневного запуска продукта.

### Viewer

Наблюдатель может смотреть dashboard, контакты, отчеты и аналитику, но не должен менять данные.

## Ежедневный процесс маркетолога

1. Открыть `/dashboard` и посмотреть блок «Фокус на сегодня».
2. Открыть `/tasks` и обработать просроченные follow-up.
3. Добавить новых мастеров/салоны в `/people`.
4. По каждому важному контакту открыть карточку и зафиксировать активность.
5. Перевести контакт по стадии в `/funnels` или прямо в карточке.
6. Отправить персональную анкету `/q/[token]`, если нужна обратная связь, привязанная к контакту.
7. В конце дня добавить инсайты или обновить гипотезы.

## Еженедельный процесс

1. Открыть `/reports`.
2. Скопировать недельный отчет для команды.
3. Проверить лучшие каналы, ниши и города.
4. Открыть `/hypotheses` и закрыть устаревшие проверки.
5. Открыть `/insights` и выбрать главные выводы недели.
6. Перед массовыми действиями скачать бэкап на `/launch`.

## Правила ведения данных

- Не создавайте контакт без источника.
- Не оставляйте контакт без следующего шага, если он важный.
- Причину отказа фиксируйте в истории контакта.
- Инсайт должен содержать доказательство, а не просто мнение.
- Гипотеза должна иметь метрику успеха.
- Кампания должна иметь оффер и итоговый вывод.
