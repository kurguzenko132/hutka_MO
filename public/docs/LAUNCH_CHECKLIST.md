# Hutka — чеклист запуска команды

## 1. Техническая готовность

- [ ] Vercel deployment имеет статус Ready.
- [ ] В Vercel настроены переменные `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `NEXT_PUBLIC_APP_URL`.
- [ ] В Supabase выполнен актуальный `supabase/schema.sql`.
- [ ] Создан минимум один пользователь с ролью `admin`.
- [ ] Страницы `/dashboard`, `/people`, `/tasks`, `/reports`, `/settings`, `/quality`, `/launch` открываются без ошибок.
- [ ] Публичная ссылка опроса `/s/[slug]` открывается без авторизации.

## 2. Настройки Hutka

- [ ] В `/settings` указано название рабочего пространства.
- [ ] Настроены реальные источники: Instagram, Telegram, рекомендации, партнеры, реклама.
- [ ] Настроены стадии воронки под текущий процесс.
- [ ] Добавлены базовые теги для сегментации контактов.
- [ ] Проверены роли пользователей: admin, marketer, viewer.

## 3. Первый рабочий сценарий

- [ ] Добавлен первый контакт в `/people/new`.
- [ ] У контакта заполнены город, ниша, источник, стадия, приоритет и следующий шаг.
- [ ] В карточке контакта добавлена активность.
- [ ] Создана задача по контакту.
- [ ] Контакт отображается в `/funnels`.
- [ ] Смена стадии отражается в истории контакта.

## 4. Опросы и кампании

- [ ] Создан опрос для мастеров.
- [ ] Проверена публичная ссылка опроса.
- [ ] Ответ на опрос сохраняется в Hutka.
- [ ] Создана первая кампания.
- [ ] К кампании привязаны контакты.
- [ ] В карточке кампании зафиксирован вывод.

## 5. Отчеты и резервные копии

- [ ] Страница `/reports` показывает недельный отчет.
- [ ] Страница `/geography` показывает города и ниши.
- [ ] Страница `/quality` не показывает критических ошибок.
- [ ] На странице `/launch` скачан первый JSON-бэкап.
- [ ] Бэкап сохранен в безопасном месте.

## 6. Правило запуска

Не начинайте массовый импорт контактов, пока не проверены: роли, справочники, создание контакта, создание задачи, публичный опрос и резервный экспорт.
